# Database Design

## Engine

PostgreSQL, hosted on Supabase. Schema is managed via Drizzle ORM
(`src/lib/db/schema/`) with migrations generated by `drizzle-kit`
(`pnpm db:generate`, `pnpm db:migrate`).

## Conventions

- Table names: `snake_case`, plural (e.g. `quotes`, `line_items`).
- Primary keys: `id` (UUID, default `gen_random_uuid()`) unless there's a
  specific reason otherwise.
- Timestamps: `created_at`, `updated_at` (`timestamptz`) on every table.
- Foreign keys: `<singular_table>_id` (e.g. `quote_id`).

## Schema

No application tables exist yet. The business domain (entities,
relationships, business rules, numbering) is fully specified in
[DOMAIN_MODEL.md](./DOMAIN_MODEL.md) — this section will document the
actual Drizzle table definitions as each entity is implemented, mapping
1:1 back to that domain model.

## Migrations

Migrations are generated from schema changes in
`src/lib/db/schema/index.ts` via `pnpm db:generate`, and applied via
`pnpm db:migrate`. Generated SQL lives in `src/lib/db/migrations/` and is
committed to version control.
